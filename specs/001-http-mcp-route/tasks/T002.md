## Context

HTTP must be disabled by default and only available through an explicit documented path. Existing stdio command behavior must remain unchanged.

Required defaults:
- `vendor/bin/kirby-mcp` remains stdio.
- HTTP mode requires an explicit subcommand/flag/config/env enablement.
- default HTTP bind host is `127.0.0.1`.
- endpoint path defaults to `/mcp`.
- auth is required when HTTP is enabled.
- shared-token mode is local/loopback development mode only.

Follow existing config style in `KirbyMcpConfig`: project config is `.kirby-mcp/mcp.json` with legacy `.kirby-mcp/config.json`, and env vars use `KIRBY_MCP_*` names.

## DoD

- Add typed HTTP config accessors or a focused HTTP config object.
- Add explicit `http` subcommand or `--transport=http` entrypoint without changing stdio defaults.
- Add config/env validation that rejects unsafe combinations such as enabled HTTP without auth material.
- Add direct Composer dependencies needed for HTTP request/response and OAuth/JWT runtime behavior.
- Add unit tests for default-disabled behavior, env/config precedence, unsafe bind/auth rejection, and shared-token local restriction.
- Add integration coverage proving normal bin commands still work.

## Validation

- `vendor/bin/pest tests/Unit/KirbyMcpHttpConfigTest.php`
- `vendor/bin/pest tests/Integration/KirbyMcpBinCommandsTest.php`
- `composer validate --strict`

## Escalate If

- The desired CLI shape conflicts with existing subcommands or client setup.
- A required dependency introduces a PHP version conflict with `php:^8.2`.

## Notes

Prefer a small explicit config surface over spreading HTTP env parsing throughout the bin file.
